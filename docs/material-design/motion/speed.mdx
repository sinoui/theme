# 速度 Speed

设置合理的动效速度可使动效过渡响应灵敏且平滑。

## 控制速度 Controlling speed

过渡速度是 UI 改变状态的可见速度。主要通过**持续时间**（过渡持续多长时间）、**缓动**（随着时间推移的加速度）和元素移动的**距离**来控制速度，即：duration, easing and distance。

在调整持续时间之前，请应用正确的缓动类型，因为缓动调整可能会影响持续时间。

### 作用 Usage

适当的过渡速度可帮助用户跟上 UI 更改，而不会使任务复杂化。应该使用尽可能短的持续时间，只要能不起到突然或震颤的反效果即可。

## 持续时间 Duration

无论采用何种动画风格，过渡都不应过快让人感觉突然，或者太慢让用户等待。正确地设置持续时间和缓动会产生流畅而清晰的过渡。以下因素详细说明了应如何调整持续时间以适应每个动画。

## 缓动 Easing

缓动是一种调整动画变化加速度的方法。例如，缓动使过渡可以加速和减速，而不是以恒定的速度移动。

在现实世界中，事情不会瞬间开始或停止。它们需要时间加速和减速。缓动会让元素像是受到摩擦力等自然力的影响一样移动。

### 缓动类型 Easing types

缓动类型可能在各个平台上的命名可能不同。在这里将它们成为标准、加速和减速。

### 标准缓动 Standard easing

**标准缓动**通过给减速比加速更多的时间，在动画的结尾处引起微妙的注意。这是最常见的缓动类型。

从静止状态开始、到静止状态结束的元素使用标准缓动。它们非常快地加速并缓慢地减速，为了强调过渡的结束。

import standardcurve from './images/standardcurve.mp4';

<ImageWithTitle video={standardcurve}>
  此图显示了一个在过渡中一直可见的元素的缓动。y轴表示位置，x轴表示时间。
</ImageWithTitle>

| 平台 | 缓动设置 |
| ---- | ------- |
| Andriod | FastOutSlowInInterpolator |
| iOS | [[CAMediaTimingFunction alloc] initWithControlPoints:0.4f:0.0f:0.2f:1.0f] |
| CSS | cubic-bezier(0.4, 0.0, 0.2, 1); |
| After Effects | Outgoing Easing: 40%, Incoming Easing: 80% |

### 减速缓动 Decelerate easing

使用减速缓动为进场元素设置动画，该过程以峰值速度（元素运动的最快点）开始过渡，并在静止时（速度为0时）结束。

import deceleratecurve from './images/deceleratecurve.mp4';

<ImageWithTitle video={deceleratecurve}>
  该图显示了仅在过渡结束时可见元素的缓动。y轴表示位置，x轴表示时间。
</ImageWithTitle>

| 平台 | 缓动设置 |
| ---- | ------- |
| Andriod | LinearOutSlowInInterpolator |
| iOS | [[CAMediaTimingFunction alloc] initWithControlPoints:0.0f:0.0f:0.2f:1.0f] |
| CSS | cubic-bezier(0.0, 0.0, 0.2, 1); |
| After Effects | Outgoing Easing: 0%, Incoming Easing: 80% |

### 加速缓动 Accelerate easing

离开屏幕的元素使用加速缓动，从静止开始并以峰值速度结束。

| 平台 | 缓动设置 |
| ---- | ------- |
| Andriod | FastOutLinearInInterpolator |
| iOS | [[CAMediaTimingFunction alloc] initWithControlPoints:0.4f:0.0f:1.0f:1.0f] |
| CSS | cubic-bezier(0.4, 0.0, 1, 1); |
| After Effects | Outgoing Easing: 40%, Incoming Easing: 0% |
