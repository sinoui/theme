### 解析过渡过程 Transition anatomy

import topappbar from './images/elementtype-topappbar.mp4';

在过渡期间，进行转换的 UI 元素分为传入、传出或永久的。元素所属的类别会影响其转换方式。

不转换的 UI 元素归为静态元素。它们在过渡期间不扮演任何角色。

<ImageWithTitle video={topappbar}>
  <h4>1. 永久元素 Persistent element</h4>

  <h4>2. 传出元素 Outgoing element</h4>
  <p>传出元素（例如标题）以淡出的方式离开屏幕。</p>
  <h4>3. 传入元素 Incoming element</h4>
  <p>传入元素（例如图标按钮），以淡入的方式进入屏幕。</p>
  <h4>4. 静态元素 Static element</h4>
  <p>静态元素（例如菜单图标），没有任何变化。</p>

## 展现连续性 Expressing continuity

当 UI 元素消失、出现、位置变动后，动效可以通过平滑、不中断的动画引用用户注意力。

一般使用以下一种或多种技术来表示连续性：

* 补间动画 Tweening
* 淡入淡出 Fading
* 元素共享动画 Shared transformation

### 补间动画 Tweening

**补间动画**显示组件或元素的变更随时间推移的平滑的进展。例如，开关可以通过补间动画实现平滑的移动；通过补间动画改变大小和圆角半径，可以让浮动操作按钮平滑地过渡为卡片。

补间动画用于有一系列中间值的转变。例如，可以让颜色从红色过渡到紫色，然后过渡到蓝色。

补间动画不能用于没有中间值的转变。如：列表布局只有整数列，1列或2列，不可能有1 ~ 2之间的列数。

import tweening from './images/tweening.mp4';

<ImageWithTitle video={tweening}>
  对浮动操作按钮的这些属性做了补间动画：位置、大小、z轴海拔、颜色、圆角半径、黑色蒙层。
</ImageWithTitle>

### 淡入淡出 Fading

**淡入淡出**是不透明度变化的补间动画，达到平滑过渡的效果。如：一个图像可以通过淡出淡入切换到另一张图像。


### 淡入淡出 Fade

淡入淡出可在完全重叠的元素之间（例如卡片或其他容器中的照片）创建平滑过渡。前景元素通过淡入淡出来显示后面一层的元素。例如：一张图片在淡出时，能看出它背后还有另一张图片。

import dissolve from './images/dissolve.mp4';

<ImageWithTitle video={dissolve}>
  一个元素的淡出，以显示隐藏在其后面的元素。
</ImageWithTitle>

### 交叉淡入淡出 Cross-fade

交叉淡入淡出说的是两个元素同时变：一个淡入，同时另一个淡出。这样在变化的过程中，二者以及它们后面的背景能够被同时看到。但如果层数太多就会分散注意力。

import crossFade from './images/cross-dissolve.mp4';

<ImageWithTitle video={crossFade}>
  尽管两个元素同时淡入淡出，但在过渡中可以同时看到这两个元素和其背景。
</ImageWithTitle>

### 依此淡入淡出 Fade through

依此淡入淡出指的是：两个元素，一个完全淡出之后，另一个才淡入。这种变化可用于文字、图标等不适合重合叠加的元素。变化过程中，下面的背景会露出来，结合元素共享动画一起使用，会提供良好的一致体验。

import fadeThrough from './images/fade-through.mp4';

<ImageWithTitle video={fadeThrough}>
  两个元素依此淡出淡入，并且在任何时候都不同时可见。
</ImageWithTitle>

### 元素共享动画 Shared transformation

**元素共享动画**通过同步的元素动作，来强化一致性。例如，如果浮动操作按钮突然显示不同的图标，可能会分散注意力。当图标处于运动状态时，共享变换可以平滑地表达图标的变化。

  </ol>
</ImageWithTitle>
