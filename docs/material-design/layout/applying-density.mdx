### 密集指南 Density guidelines

#### 如何应用 How to apply

Material Design 密集指南是帮助理解何时以及如何应用密集以改进用户体验的一份资料。实现 Material Design 的 UI 库一般会支持密集模式定制。

#### 何时应用 When to apply

在设计用户与组件交互时，应确定是否增加用户界面的密度。当用户查看、处理大量信息时，高密度组件可以改善体验。

通过在屏幕上提供更多内容，可以增加列表、表格、长表单的密度，从而使信息更易于扫描、查看和比较。

import apply from './images/density-apply.mov';

<ImageWithTitle img={apply}>
  <ol>
    <li>增加数据表格的密度。</li>
    <li>增加表单的密度。</li>
  </ol>
</ImageWithTitle>

### 何时不使用密度 When not to apply density

任务和基于警报的组件不应增加密度。

#### 重点任务 Focused tasks

不要增加包含重点任务的组件密度，如下拉菜单、选择器。诸如日期选择器组件，增加了密度，减少了可点击的空间，从而降低了可用性。

import focusedTasksDont from './images/focused-tasks-dont.png';

<ImageWithTitle status="no" img={focusedTasksDont}>
  不要增加包含重点任务的组件密度，如日期选择器。
</ImageWithTitle>

#### 警告和消息 Alerts and messaging

不要增加提醒用户的组件密度，如对话框。警报类组件的密度增加使它们吸引用户注意力方面效果不佳。在对话框的示例中，密度增加会损害消息的可读性和突出性。

import alertsDont from './images/alerts-dont.png';

<ImageWithTitle status="no" img={alertsDont}>
  不要增加警告和消息组件的密度，如对话框。
</ImageWithTitle>

## 布局 Layout

### 网格和组件密度 Grid and component density

维护组件密度和网格布局之间的平衡。

要创建可扫描的内容组，请结合使用低密度的网格布局和高密度的组件。你的组件密度越高，网格的边距和间距应越大。高密度布局网格和高密度组件一块使用会让用户难以使用。

import gridDo from './images/gridvcontent-do.mov';
import gridDont from './images/gridvcontent-dont.mov';

<ImageWithTitle status="ok" img={gridDo}>
  高密度组件与低密度边距、间距组合使用。
</ImageWithTitle>
<ImageWithTitle status="no" img={gridDont}>
  不要同时使用高密度组件和布局网格。
</ImageWithTitle>

## 组件 Components

### 给组件应用密度 Applying density to components

不要单独调整组件密度。相反，应为组件集使用密度。当你调整一个组件的密度，需考虑这个变更是否会影响到您的应用程序中其它的组件。

Material 组件有三个密集维度：默认（default）, 舒适（comfortable）, 紧凑（compact）。

### 密度刻度 Density scale

密度刻度使您在需要时控制各个组件的密度。

密度刻度已编号，从 0 开始表示组件的默认密度。随着空间的减小，密度刻度将移至负数（-1, -2, -3），从而创建更高的密度。每个增量代表组件高度减少 4dp。

import buttonDensityScale from './images/button-density-scale.mov';
import chipDensityScaleDont from './images/chip-density-scale-dont.mov';

<ImageWithTitle img={buttonDensityScale}>按钮应用上密度刻度。</ImageWithTitle>
<ImageWithTitle img={chipDensityScaleDont} status="no">
  不同因增加密度太多而破坏了组件。chip组件使用了 -4 密度后，文字溢出了。
</ImageWithTitle>

## 可访问性 Accessibility

### 触摸目标 Touch targets

触摸目标适用于同时接收触摸输入和非触摸输入（如鼠标点击）的任何设备。为了平衡信息密度和可用性，触摸目标至少为 48 x 48 dp，目标之前至少要有 8dp 的间距。

import touchtargets from './images/layout-spacing-touchtargets.png';

<ImageWithTitle img={touchtargets}>
  <p>触摸目标最小尺寸： 48 x 48 dp。</p>

  <ol>
    <li>40dp 元素，48dp 触摸目标</li>
    <li>24dp 元素，48dp 触摸目标</li>
    <li>36dp 元素，48dp 触摸目标</li>
  </ol>
</ImageWithTitle>

48 x 48 dp的最小触摸目标应是您产品中默认的体验设置。用户必须选择高密度组件体验，因为更高密度的组件无法满足可访问性要求。

  用户选择加入更密集的组件体验模式。
</ImageWithTitle>
