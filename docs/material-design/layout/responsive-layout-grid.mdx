# 响应式布局网格 Responsive layout grid

Material Design 响应式布局网格可适应屏幕尺寸和方向，从而确保跨布局的一致性。

## 列，间距和边距 Columns, gutters, and margins

Material Design 布局网格由三个元素组成：列、间距和边距。

### 列 Columns

内容放置在包含列的屏幕区域中。

列宽是使用百分比而不是固定值来定义的，以允许内容灵活地适应任何屏幕大小。网络中显示的列数由查看屏幕的断点范围（预定屏幕大小的范围）确定，无论是移动设备、平板电脑还是其他尺寸的断点。

</ImageWithTitle>
<ImageWithTitle img={columns600}>

</ImageWithTitle>

### 间距 Gutters

间距是列之间的空间。它们有助于分离内容。

间距宽度在每个断点范围内的值是固定的。为了更好地适应屏幕，间距宽度可以在不同的断点处改变。更宽的间距宽度更适合较大的屏幕，因为它们在列之间创建了更多的空白。

### 边距 Margins

边距是内容与屏幕左右边缘之间的空间。

边距宽度在每个断点范围内的值是固定的。为了更好地适应屏幕，边距宽度可以在不同的断点处改变。更宽的边距宽度更适合较大的屏幕，因为它们会在内容周围产生更多的空白。

### 水平网格 Horizontal grids

可以针对水平滚动的触摸 UI 自定义 Material Design 布局网格。列、间距和边距是从左到右放置的，而不是从上到下。屏幕的高度确定水平网格中的列数。

在非触摸和 Web 平台上，水平滚动 UI 并不常见。

## 断点 Breakpoints

断点是具有特定布局要求的预定屏幕大小的范围。在给定的断点范围内，布局将根据屏幕大小和方向进行调整。

<ImageWithTitle video={breakpoints} />

### 断点系统 Breakpoint system

Material Design 根据以下列结构提供响应式布局。4列、8列和12列网格布局可用于不同的屏幕、设备和方向。

每个断点范围确定每种显示尺寸的列数以及建议的边距和间距。

| 断点范围(dp) |      竖向     |       横向       |  窗口   |  列 | 边距/间距 |
|-------------|---------------|------------------|--------|-----|-----------|
| 0 - 359     | small handset |                  | xsmall |  4  |     16    |
| 360 - 399   | medium handset |                 | xsmall |  4  |     16    |
| 400 - 479   |  large handset |                 | xsmall |  4  |     16    |
| 480 - 599   | large handset  | small handset   | xsmall |  4  |     16    |
| 600 - 719   | small tablet   | medium handset  | small  |  8  |     16    |
| 720 - 839   | large tablet   | large handset   | small  |  8  |     24    |
| 840 - 959   | large tablet   | large handset   | small  | 12  |     24    |
| 960 – 1023  |                | small tablet    | small  | 12  |     24    |
| 1024 – 1279 |                | large tablet    | medium | 12  |     24    |
| 1280 – 1439 |                | large tablet    | medium | 12  |     24    |
| 1440 – 1599 |                |                 | large  | 12  |     24    |
| 1600 – 1919 |                |                 | large  | 12  |     24    |
| 1920 +      |                |                 | xlarge | 12  |     24    |

## 网格行为 Grid behavior

### 弹性网格 Fluid grids

弹性网格使用可缩放和调整内容大小的列。弹性网格的布局可以使用断点来确定布局是否需要显著更改。

import fluid from './images/layout-responsive-grid-behavior-fluid.gif';

<ImageWithTitle img={fluid}>
  列在全宽网格中扩展
</ImageWithTitle>

### 固定网格 Fixed grids

固定网格使用固定大小的列，具有弹性边距，以在每个断点范围内保持内容不变。固定网格的布局只能在指定的断点处更改。

<ImageWithTitle img={fixed}>
  边距在固定网格中扩展
</ImageWithTitle>

## 用户界面区域 UI regions

布局由多个UI区域组成，例如侧面导航，内容区域和应用栏。这些区域可以显示操作、内容或导航目标。UI区域应跨设备保持一致，同时适应不同屏幕尺寸的不同断点。

为了提高设备之间的熟悉度，为桌面设计的UI元素应该以与移动UI一致的方式进行组织。

<ImageWithTitle img={uiRegions}>
  在不同尺寸的屏幕上更改布局
</ImageWithTitle>

### 永久用户界面区域  Permanent UI regions

永久用户界面区域是可以在响应网格之外显示的区域，如导航抽屉。这些地区不能重叠。

import permenant from './images/layout-responsive-uiregions-permenant.png';

<ImageWithTitle img={permenant}>
  当屏幕空间可用时，永久用户界面区域将显示内容。
</ImageWithTitle>

### 持久用户界面区域 Persistent UI regions

持久性用户界面区域是可以随时根据命令显示的区域，或者它们可以保持可见。它们可以打开或关闭，出现或消失。当它们出现时，它们会压缩内容和网格。

当持久性用户界面区域可见时，其可见性不受屏幕上其他元素交互的影响。

import persistent from './images/layout-responsive-uiregions-persistent.gif';

<ImageWithTitle img={persistent}>
  当打开时，这个持久层导航抽屉会使网格（及其内容）压缩。
</ImageWithTitle>

### 临时用户界面区域 Temporary UI regions

临时用户界面区域暂时出现，当它们出现时，它们不会影响响应式网格。当它们可见时，可以通过点击其区域中的项目或其区域外的任何空间来隐藏它们。

当用户界面区域可见时，其他屏幕元素不是交互式的。

<ImageWithTitle img={temporary}>

</ImageWithTitle>
