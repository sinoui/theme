# 组件行为 Component behavior

以下指南描述了组件在网格中和断点之间的行为方式。

## 定位术语 Positioning terminology

用户界面元素在每个断点范围内的行为方式不同。

### 术语解释 Descriptors

用户界面元素、组件和表面在网格中的位置可以由下面的术语描述：

| 术语         | 定义                                             |
| ------------ | ------------------------------------------------ |
| 上方, 下方   | 一个元素的 y 轴位置                              |
| 前方，后方   | 一个元素的 z 轴位置                              |
| 左，右，居中 | 一个元素 x 轴位置                                |
| 顶部，底部   | 某元素相对于容器或者屏幕边缘的 y 轴位置          |
| 垂直居中     | 某元素的 x 轴和 y 轴删位置都位于容器或屏幕的中心 |
| 粘性定位     | 跟随用户界面滚动并锁定在滚动位置上的某个点       |
| 浮动         | 固定在滚动内容前面的元素位置                     |

## 组件宽度 Component width

组件宽度可以在整个屏幕尺寸上保持不标，也可以根据布局而变化。组件宽度可以是：

- 固定的
- 弹性的

由于固定宽度元素不会在布局变化时保持宽度不变，因此可以更改它们的位置以适应新布局。它们的位置可以是：

- 推入的
- 覆盖的

### 推入 Pushing

当布局和屏幕尺寸发生变化时，组件可以从它原始位置推入出来。组件的宽度保持不变，但是它的水平位置发生变化，向左或者向右移动。当发生这种情况时，如果被推离屏幕，它可能会被屏幕边缘部分遮挡。

<ImageWithTitle img={push}>导航抽屉将内容区域推离屏幕。</ImageWithTitle>

### 覆盖 Overlaid

当用户界面尺寸发生变化时，一个组件可能会被其它遮挡它的组件覆盖。当一个组件被覆盖，它的宽度和位置保持不变。

<ImageWithTitle img={overlaid}>内容区域被导航区域覆盖。</ImageWithTitle>

## 响应模式 Responsive patterns

组件可以使用以下模式根据屏幕尺寸和设备类型调整其尺寸：

- 使用不同组件以更好地适应空间
- 更改组件的可见性以容纳较小的空间
- 更改输入的方法

随着屏幕空间的增长，可以应用以下响应模式。

当在指定的断点处有更多空间可用时，可以显示较小屏幕所隐藏的部分 UI。
